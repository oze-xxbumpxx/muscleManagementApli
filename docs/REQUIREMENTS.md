# 要件定義書

muscleManagementApli - 筋力トレーニング管理アプリ

---

## プロダクト概要

本プロダクトは、個人利用を目的とした筋力トレーニング管理アプリである。

5年以上筋力トレーニングを継続しているが、トレーニング内容のマンネリ化や、
成長・継続状況が可視化されていないことによるモチベーション低下を課題としている。

トレーニングの記録・振り返り・可視化を行うことで、
トレーニングの効果向上と継続を支援することを目的とする。

---

## 想定ユーザー

- **対象**：自分自身（個人利用）
- **年齢層**：20代
- **トレーニング歴**：5年以上
- **ITリテラシー**：Webアプリの利用・開発に慣れている
- **利用シーン**：
    - トレーニング中（記録・タイマー）
    - トレーニング終了後（記録・振り返り）

---

## 解決したい課題

### 現状の課題

1. **マンネリ化**
   - 同じ種目ばかりになりがち
   - バリエーションが増えない

2. **記録の不在**
   - 過去のトレーニングが体系的に管理されていない
   - どの種目をいつやったか思い出せない

3. **モチベーション低下**
   - 継続や成長が可視化されていない
   - 頑張りが目に見えない

---

## MVP1の目的

筋力トレーニングを

**「記録できる・振り返れる・トレーニング中も使える」**

状態を実現し、マンネリ化とモチベーション低下を防ぐ。

---

## MVP1 機能要件

### 1. トレーニング記録機能

**目的**：日々のトレーニング内容を記録する

**機能詳細**：
- 日付単位でトレーニングを管理
- 1日に複数種目を登録可能
- 各種目に以下の情報を紐付ける：
    - 種目名（例：ベンチプレス、スクワット）
    - 重量（kg）
    - 回数（rep）
    - セット数
- CRUD操作が可能
    - **Create**：新規トレーニング記録作成
    - **Read**：記録の閲覧
    - **Update**：記録の編集
    - **Delete**：記録の削除

**データ構造イメージ**：
```
TrainingSession (日付単位)
  ├── date: 2026-01-21
  └── exercises: [
        {
          name: "ベンチプレス",
          weight: 80,
          reps: 10,
          sets: 3
        },
        {
          name: "スクワット",
          weight: 100,
          reps: 8,
          sets: 3
        }
      ]
```

---

### 2. 記録一覧・詳細表示機能

**目的**：過去のトレーニングを振り返る

**機能詳細**：
- 日付順でトレーニング記録の一覧を表示
- 日付を選択すると、その日の詳細を表示
- 詳細画面では：
    - 実施した全種目
    - 各種目の重量・回数・セット数

**UI イメージ**：
- 一覧：カード形式 or リスト形式
- 詳細：種目ごとに整理された表示

---

### 3. 種目履歴可視化機能

**目的**：マンネリ化を防ぐために、最近の種目傾向を把握する

**機能詳細**：
- **直近5回の種目一覧**
    - 最近実施した5回のトレーニングで行った種目を表示
    - 偏りがあれば気づける

- **同一種目の連続実施回数**
    - 特定の種目を連続で何回やっているかを表示
    - 例：「ベンチプレスを連続5回実施中」

**表示例**：
```
直近5回の種目：
- ベンチプレス（3回）
- スクワット（2回）
- デッドリフト（1回）

連続実施：
- ベンチプレス：5回連続
```

---

### 4. 成長可視化機能

**目的**：成長を実感し、モチベーションを維持する

**機能詳細**：
- 種目ごとの折れ線グラフ表示
- X軸：日付
- Y軸：重量 または 回数（切り替え可能）
- 複数種目の比較も可能

**グラフイメージ**：
```
重量の推移（ベンチプレス）

100kg |                    ●
      |                 ●
 80kg |           ●
      |        ●
 60kg |     ●
      |________________
      1月  2月  3月  4月
```

---

### 5. 継続可視化機能

**目的**：継続を実感し、習慣化を促進する

**機能詳細**：

**5-1. カレンダー表示**
- 月単位のカレンダー
- トレーニング実施日をマーク
- 実施日をクリックすると詳細へ遷移

**5-2. ストリーク表示**
- 現在の連続トレーニング日数
- 最長連続記録
- 今月の実施日数

**表示例**：
```
📅 今月の実施状況：12/31日
🔥 現在のストリーク：7日連続
🏆 最長ストリーク：21日連続
```

---

### 6. タイマー機能（Front完結）

**目的**：トレーニング中のインターバル管理

**機能詳細**：
- 種目ごとに1つタイマーを設定可能
- 操作：
    - 開始（Start）
    - 停止（Pause）
    - リセット（Reset）
- カウントダウン形式
- アラート機能（音 or 通知）

**実装方針**：
- フロントエンド完結（Tactile State）
- ViewModel層で管理

**UIイメージ**：
```
インターバルタイマー
  02:30
[開始] [停止] [リセット]
```

---

## MVP1 で やらないこと

以下の機能はMVP1では実装しない：

- ❌ **認証 / マルチユーザー**
    - 個人利用のため不要
    - 将来的に追加可能性あり

- ❌ **SNS共有機能**
    - 記録を共有する必要なし

- ❌ **栄養管理**
    - トレーニングに集中

- ❌ **レコメンド機能**
    - 種目提案などは手動で判断

- ❌ **AI要素**
    - 複雑化を避ける

- ❌ **E2Eテスト**
    - ユニット・統合テストに注力

- ❌ **デプロイ最適化**
    - ローカル開発優先
    - 将来的にAWSデプロイ予定

## 非機能要件

### パフォーマンス
- 記録表示：1秒以内
- グラフ描画：2秒以内

### 可用性
- ローカル環境で動作
- オフライン対応は不要

### セキュリティ
- 個人利用のため最小限
- SQLインジェクション対策（Sequelize利用）
- XSS対策（React標準機能）

### 保守性
- クリーンアーキテクチャ採用
- テストカバレッジ重視
- ドキュメント整備
